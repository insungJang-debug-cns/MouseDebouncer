# 더블클릭 방지

마우스 버튼 채터링(이중 클릭) 방지 유틸리티. 설정한 딜레이(ms) 이내의 재입력을 차단합니다.

## 설치 및 실행

별도 설치 불필요. `MouseDebouncer.exe`를 실행하면 시스템 트레이에 상주합니다.

> .NET 8 런타임이 없는 PC에 배포할 때는 단일 exe 퍼블리시 빌드를 사용하세요.  
> (`dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true`)

## 기능

### 설정 창

트레이 아이콘 더블클릭 또는 우클릭 → **설정 열기** 로 열 수 있습니다.

| 항목 | 설명 |
|---|---|
| 사이드 버튼 1 딜레이 (ms) | 마우스 사이드 버튼 1 딜레이 (기본값: 300ms) |
| 사이드 버튼 2 딜레이 (ms) | 마우스 사이드 버튼 2 딜레이 (기본값: 300ms) |
| 왼쪽 버튼 딜레이 (ms) | 왼쪽 버튼 딜레이 (기본값: 0, 비활성) |
| 오른쪽 버튼 딜레이 (ms) | 오른쪽 버튼 딜레이 (기본값: 0, 비활성) |
| 휠 클릭 딜레이 (ms) | 휠 클릭 딜레이 (기본값: 0, 비활성) |
| 시작 프로그램 등록 | Windows 시작 시 자동 실행 여부 |
| 누른 버튼 | 설정 창이 열려 있을 때 마우스 버튼을 누르면 어떤 버튼인지 표시 |
| 더블클릭 방지횟수 | 오늘 차단된 클릭 수 (자정 리셋) |

- 딜레이를 `0`으로 설정하면 해당 버튼 비활성화
- 단위: ms **(1000 = 1초)**
- **저장** 버튼을 눌러야 설정이 저장됩니다

#### 내 마우스 버튼 확인 방법

설정 창을 열어둔 상태에서 확인하고 싶은 버튼을 누르면 **"누른 버튼:"** 항목에 버튼 이름이 표시됩니다.  
해당 이름의 딜레이 항목에서 값을 조정하세요.

### 트레이 아이콘

| 메뉴 | 설명 |
|---|---|
| 설정 열기 | 설정 창 열기 |
| 비활성화 / 활성화 | 디바운싱 일시 중지 / 재개 |
| 오늘 방지횟수 | 오늘 차단된 클릭 수 |
| 로그 기록 시작 / 중지 | 클릭 이벤트 로그 기록 토글 |
| 로그 파일 열기 | 기록된 로그를 메모장으로 열기 |
| 종료 | 프로그램 종료 |

### 로그 기록

뚫리는 현상 진단 시 사용합니다.

1. 트레이 우클릭 → **로그 기록 시작** (기존 로그 자동 초기화)
2. 문제가 발생할 때까지 마우스를 사용
3. 트레이 우클릭 → **로그 파일 열기**

로그 예시:
```
[15:30:01.234] 사이드 버튼 1 | 통과  | 경과: 첫 입력
[15:30:01.280] 사이드 버튼 1 | 차단  | 경과:     46ms | 딜레이: 300ms
[15:30:01.590] 사이드 버튼 1 | 통과  | 경과:    356ms | 딜레이: 300ms  ← 딜레이 초과로 뚫림
```

버튼 DOWN 이벤트가 차단되면 대응하는 UP 이벤트도 함께 차단합니다.  
(브라우저 등 일부 앱이 UP 이벤트에 반응하여 이중 동작하는 문제 방지)

로그 파일 위치: 실행 파일과 같은 폴더의 `debounce_log.txt`

- 설정 창의 X 버튼은 종료가 아닌 트레이로 최소화합니다
- 비활성 상태일 때 트레이 아이콘이 방패 → 경고 아이콘으로 변경됩니다

## 설정 파일

`config.json` — 실행 파일과 같은 폴더에 자동 생성됩니다.

```json
{
  "Left": 0,
  "Right": 0,
  "Middle": 0,
  "XButton1": 300,
  "XButton2": 300
}
```
